# Style Guide

- Use black-and-white academic block-diagram style.
- Use 3 pt strokes for visible signal lines and component outlines.
- Use orthogonal routing only unless the user explicitly requests otherwise.
- Prefer direct connections, then one-bend routes, then outer lanes for long feedback.
- Keep main signal chains aligned on a shared centerline.
- Use larger bold labels for external signals such as `X(z)`, `Y(z)`, and `E(z)`.
- Keep gain labels horizontal even when the gain triangle is oriented left, up, or down.
- Use component catalog templates for standard components.
